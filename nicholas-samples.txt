    1. What is Semantic HTML? 
Semantic HTML is a code tag that divides a webpage into sections to help the developer stay organized, and allow them to later manipulate the various sections in CSS
    2. What is HTML used for? 
HTML is a markup language used to write the basic structure of a web page.
    3. What is an attribute and where do we put it? 
An attribute provides more info, such as a 'src' attibute in an 'img' tag, which tells the source of the image file we want to call.
    4. What is the h1 tag used for? How many times should I use it on a page?
The h1 tag is for the 'title' of the webpage, or the primary heading, and should only be used once!
    5. Name two tags that have required attributes
The <img> tag requires a src, and the <a> tag requires a href.
    6. What do we put in the head of our HTML document? 
The head section of the document is for metadata about the document itself, it does not get displayed on the page.
    7. What is an id? 
A tag created by the developer that can later be called in CSS to manipulate a specific element.
    8. What elements can I add an id to? 
Any element in HTML
    9. How many times can I use the same id on a page? 
Only once!
    10. What is a class? 
Similar to an id, a developer can give elements a class name, to help organize and group things together.  The developer can later call all elements in the same class and manipulate them, rather than needing to call each element individually.
    11. What elements can I add a class to? 
Any element in HTML
    12. How many times can I use the same class on a page? 
There is no limit to how many elements can be put into a class
    13. How do I get my link to open in a new tab?
Using <...target="_blank">
    14. What is the alt attribute in the image tag used for? 
It is a text description of the image, used by search engine bots and also screen-readers to tell what the image is on your page
    15. How do I reference an id?
Using #idName in CSS
    16. What is the difference between a section and a div
Sections divide up the whole page, while divs should be used to divide elements within sections
    17. What is CSS used for? 
To style HTML and make it look pretty on a page!
    18. How to we select an element? Example - every h2 on the page
By calling "h2 {<your code>}" in CSS
    19. What is the difference between a class and an id? - Give me an example of when I might use each one
I would use id for example by calling the header of the page "myHeader" when I only need to call that one element.  I would use a class, say, if I wanted a <h2>, a <p>, and a <ul> in the middle of my page to behave the same way, I could give them all the Class "middleText" and be able to change them all at once.
    20. How do we select classes in CSS?
Using .className in CSS
    21. How do we select a p element with a single class of “human””?
p.human
    22. What is a parent child selector? When would this be useful? 
It lets me select an elements within a larger section.  It's useful if I want to manipulate only the <p> element in one section, rather than ALL <p> elements on the page.
    23. How do you select all links within a div with the class of sidebar?
.sidebar:link
    24. What is a pseudo selector?
An example is the ":hover" selector.  So it is a selector for a specific state of the element, such as when the mouse hovers over it
    25. What do we use the change the spacing between lines?
line-height
    26. What do we use to change the spacing between letters?
letter-spacing
    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
text-transform:uppercase; text-transform:lowercase; text-transform:capitalize;
    28. How do I add a 1px border around my div that is dotted and black?
div {border: 1px dotted black;}
    29. How do I select everything on the page? 
the * selector
    30. How do I write a comment in CSS?
using /* ... */
    31. How do I find out what file I am in, when I am using the command line? 
using the 'pwd' command
    32. Using the command line - how do I see a list of files/folders in my current folder?
using the 'ls' command
    33. How do I remove a file via the command line? Why do I have to be careful with this? 
using the 'rm' command.  Be careful because deleting is permanent, and you aren't prompted "are you sure?" first.
    34. Why should I use version control? 
it makes it much easier to fix if your changes break your code, as you are able to easily go back to a previous version
    35. How often should I commit to github?
often!  whenever changes are made that you don't want to lose.
    36. What is the command we would use to push our repo up to github? 
git push -u origin "..."
